16 Mar 2021

如何恢复你的 bashrc

如果你不慎删除了你的 bashrc，或者遇到了如下的情况：

  • 在增加环境变量的时候手贱把 < 打成了 <<
  • 不慎在备份 bashrc 的时候搞错了操作数的方向

请你不要马上关闭现有的 shell。许多有价值的信息可以从其中恢复。

环境变量

env
# SHELL=/bin/bash
# SESSION_MANAGER=local/unix:@/tmp/.ICE-unix/2668,unix/unix:/tmp/.ICE-unix/2668
# ...

函数

declare -F
# declare -f j
# declare -f jc
# declare -f unproxy
# ...

执行得到的命令，就可以打印出函数的声明。

别名

alias
# ...
# alias tr='todo remove'
# alias typora='/opt/typora/Typora'
# alias vim='nvim'
# alias which='(alias; declare -f) | /usr/bin/which --tty-only --read-alias --read-functions --show-tilde --show-dot'
# alias xzegrep='xzegrep --color=auto'
# alias xzfgrep='xzfgrep --color=auto'
# alias xzgrep='xzgrep --color=auto'
# ...

系统的 bashrc 模板

cat /etc/skel/.bashrc

不能恢复的东西

source ... 语句，和代码的逻辑。

祝你好运！
